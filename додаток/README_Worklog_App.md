# Work Is Easy / Direct_hoofdrapport

## Опис проєкту

Це мобільний Android-додаток, створений у **MIT App Inventor**, для **розпізнавання мовлення** та **збереження історії розпізнаних фраз**.

Користувач обирає мову зі списку, натискає кнопку запису, після чого додаток запускає системне розпізнавання голосу. Отриманий текст автоматично зберігається разом із датою та часом. Окремий екран дозволяє переглядати всю історію записів і видаляти непотрібні елементи.

## Основна ідея

Додаток виконує роль простого голосового журналу / speech-to-text блокнота:

- вибір мови розпізнавання;
- запуск голосового вводу;
- автоматичне збереження результату;
- перегляд усіх попередніх записів;
- видалення окремих записів.

## Технологія

Проєкт зібраний у форматі **`.aia`** для **MIT App Inventor**.

Усередині проєкту є:

- `Screen1` — головний екран;
- `Screen2` — екран історії;
- `SpeechRecognizer` — для розпізнавання мовлення;
- `TinyDB` — для локального збереження даних;
- `Clock` — для запису дати та часу;
- графічні ресурси (`png`) для кнопок і фону.

## Структура екранів

### Screen1 — головний екран

Головний екран відповідає за:

- ініціалізацію додатка;
- завантаження збережених записів із `TinyDB`;
- завантаження раніше вибраної мови;
- відображення списку доступних мов у `Spinner`;
- запуск розпізнавання мовлення;
- перехід до екрана історії.

#### Компоненти Screen1

- **Spinner1** — список мов для вибору;
- **Button_Record** — кнопка запуску розпізнавання;
- **Button_View** — кнопка переходу до історії;
- **SpeechRecognizer1** — модуль speech-to-text;
- **TinyDB1** — локальна база збереження;
- **Clock1** — формування дати та часу;
- фон і графічні кнопки через `goland.png`, `baton1.png`, `batton2.png`.

### Screen2 — екран історії

Другий екран показує всі раніше збережені записи.

#### Компоненти Screen2

- **ListView1** — список усіх записів;
- **Button_Back** — повернення на головний екран;
- **Button_Delete** — видалення вибраного запису;
- **TinyDB1** — читання та оновлення історії;
- **Clock1** — службовий компонент, присутній у проєкті.

## Як працює логіка додатка

### 1. Запуск додатка

Під час відкриття `Screen1` додаток:

1. Формує список мов для відображення в `Spinner`.
2. Запитує дозвіл на використання аудіо / мікрофона.
3. Завантажує з `TinyDB`:
   - список записів (`records`);
   - лічильник ID (`idCounter`);
   - останню вибрану мову (`selected_language`).
4. Якщо мова вже була вибрана раніше, автоматично встановлює її у списку.

### 2. Вибір мови

Коли користувач змінює мову в `Spinner`:

- додаток знаходить відповідний мовний код;
- записує цей код у глобальну змінну;
- зберігає вибір у `TinyDB`, щоб використати його при наступному запуску.

### 3. Розпізнавання голосу

Після натискання **Record**:

1. Обрана мова передається у `SpeechRecognizer.Language`.
2. Запускається `SpeechRecognizer.GetText`.
3. Після завершення розпізнавання спрацьовує подія `AfterGettingText`.

### 4. Збереження результату

Після отримання тексту додаток:

1. Збільшує внутрішній лічильник записів.
2. Формує новий запис у структурі:
   - `ID`
   - `Дата`
   - `Час`
   - `Розпізнаний текст`
3. Додає новий запис на початок списку.
4. Зберігає оновлений список у `TinyDB`.
5. Зберігає нове значення лічильника.

Тобто нові записи відображаються як найсвіжіші.

### 5. Перегляд історії

Після натискання **View** відкривається `Screen2`.

На цьому екрані:

- із `TinyDB` зчитується весь список записів;
- формується список для відображення у вигляді:

`дата - час: текст`

### 6. Видалення запису

Коли користувач обирає елемент у `ListView` і натискає **Delete**:

- вибраний запис видаляється зі списку;
- оновлений список зберігається назад у `TinyDB`;
- список на екрані автоматично оновлюється.

### 7. Повернення на головний екран

Після закриття `Screen2` головний екран повторно підтягує список записів із `TinyDB`, щоб стан даних був актуальним.

## Які дані зберігаються

Додаток використовує `TinyDB` з namespace `RecordsDB`.

Зберігаються такі ключі:

- **`records`** — список усіх записів;
- **`idCounter`** — лічильник записів;
- **`selected_language`** — останній вибраний мовний код.

## Формат одного запису

Кожен запис зберігається як список із 4 елементів:

1. `ID`
2. `Дата` у форматі `dd.MM.yyyy`
3. `Час`
4. `Розпізнаний текст`

## Що вже реалізовано добре

- проста й зрозуміла логіка;
- локальне збереження без сервера;
- запам’ятовування мови між сесіями;
- окремий екран історії;
- можливість видаляти записи;
- нові записи додаються зверху, що зручно для користувача.

## Поточні обмеження

У поточній версії я бачу такі обмеження:

- немає експорту записів у файл;
- немає пошуку по історії;
- немає редагування запису;
- немає синхронізації з хмарою;
- немає сортування або фільтрації;
- немає підтвердження перед видаленням;
- немає окремого перегляду повного тексту, якщо запис довгий.

## Для чого підходить цей додаток

Цей проєкт можна позиціонувати як:

- voice notes app;
- speech-to-text notebook;
- personal work log / voice journal;
- простий мобільний інструмент для швидкої фіксації фраз голосом.

## Короткий технічний підсумок

Проєкт реалізує базовий, але повністю робочий цикл:

**вибір мови → запис голосу → розпізнавання → автозбереження → перегляд історії → видалення запису**.

Для MIT App Inventor це акуратна й логічна структура з правильним поділом на головний екран та екран перегляду історії.
